# Generate Description

Write meta descriptions for markdown content by extracting from the post — not summarizing above it.

The worst descriptions are generated *about* a post. The best are extracted *from* it. This distinction is the core of the skill.

## Inputs

1. **One or more markdown files** — with YAML frontmatter
2. **Mode** — `generate` (missing descriptions), `audit` (evaluate existing), or `both`
3. **Scope** — file glob, collection path, or explicit file list

## Outputs

For each file: a proposed `description` value ready to insert into frontmatter. In audit mode, also flags problems with existing descriptions.

Present results as a table for operator review before applying.

## Process

### Step 1: Classify search intent

Read the post and classify it into one of two categories:

**Searchable** — Someone would Google a problem and this post answers it. Technical tutorials, reference guides, how-tos, tool reviews, comparisons.
- Signal: code blocks, step-by-step structure, tool/API names, "how to" framing

**Shareable** — Someone encounters this through a link, social share, or browsing. Personal essays, reflections, found notes, life stories, rants.
- Signal: first-person narrative, no code, aphoristic title, confessional tone

This classification determines the extraction method.

### Step 2: Extract the description

#### For searchable posts

1. Identify the primary search query — what would someone type into Google to find this?
2. Lead the description with the topic keyword or problem statement
3. Include specific terms: API names, tool names, the exact problem being solved
4. Close with what the reader gets: examples, comparison, reference, step-by-step

**Pattern:** `[Topic/problem]. [What the post covers]. [What the reader gets].`

**Example:**
- Search query: "nullish coalescing vs logical or"
- Description: `Nullish coalescing (??) and logical OR (||) differ when the left side is falsy but not null or undefined. Examples with numbers, strings, and booleans.`

#### For shareable posts

1. Find the most compelling line or image from the post itself
2. Pull it verbatim or near-verbatim — preserve the author's voice
3. If the post is very short (found note, seed), the description may be the entire post
4. Do not genericize. Do not add moral framing the author didn't write.

**Pattern:** Pull the hook. Use the author's words.

**Example:**
- Post opens with: "My existential dread is a morning person."
- Description: `My existential dread is a morning person. So I accept defeat. The day is already lost. And I make my bed — my first act of defiance against the day.`

### Step 3: Validate constraints

- **Length:** 150-155 characters for Google snippet display. Shorter is fine. Longer gets truncated.
- **Accuracy:** The description must match what the post actually says. Read the post, not just the title.
- **Self-contained:** The description should make sense without the title. Google sometimes renders them independently.
- **No trailing ellipsis or cliffhangers** unless the post itself uses them.

### Step 4: Audit existing descriptions (audit mode)

Read the existing `description` field against the post content. Flag:

1. **Content mismatch** — description describes a different topic than the post covers
2. **AI-isms** — opens with "Discover", "Explore", "Embrace", "Unlock", "Master", "Dive into", "Learn the secrets of"
3. **Generic self-help** — could describe any post: "achieve your goals", "transform your productivity", "embrace your potential"
4. **Too long** — over 160 characters (Google truncation)
5. **Empty or broken** — empty string, multiline YAML accident, or missing field
6. **Typos** — in the description itself (distinct from intentional style in the post)

For each flagged description, propose a replacement using the extraction method above.

## Anti-patterns

Descriptions generated by an LLM about a post (rather than extracted from it) follow predictable patterns. Reject descriptions that:

- **Open with imperative verbs aimed at the reader**: "Discover", "Explore", "Learn", "Master", "Unlock", "Embrace", "Dive into"
- **Moralize**: "turning pain into valuable lessons", "embrace imperfection and watch your life improve"
- **Genericize**: replacing a specific story with a universal platitude. If the post is about a $9 cleaver and a dead father, the description should mention a $9 cleaver and a dead father — not "the importance of everyday tools in creating lasting memories"
- **Inflate**: "Unlock the power of Role-Based Access Control" when the post just shows how to set up RBAC
- **Add framing the author didn't write**: "Can making the bed be an act of defiance?" when the author states it directly: "It's my first act of defiance against the day"

The test: **could this description have been written by someone who only read the title?** If yes, it's bad. A good description proves the writer read the post.

## Tagging

When generating descriptions, also apply tags from the site's existing vocabulary.

**How to discover the vocabulary depends on the site.** The calling skill or CLAUDE.md should provide a query command. The general approach:
1. Scan existing tags in the content directory to get the live vocabulary with frequency counts
2. Apply 1-3 tags per post from the existing vocabulary
3. Prefer specific over general
4. Do not invent new tags unless the new tag would apply to 3+ posts
5. If nothing fits, the post doesn't need tags

Tags are low-risk and reversible. Apply them without asking.

## Composition

This skill provides the technique for description and tag generation. It is designed to be called by site-specific skills that know where content lives and what frontmatter conventions to follow.

Typical callers:
- Site-specific `publish-post` skills
- Video publish workflows
- Content platform skills (`content-linkedin`, `content-twitter`)
- Batch enrichment passes
- Shutdown/triage rituals

## Calibration

These values are current best-known and may change:

- Google snippet display: ~155 characters (as of 2025)
- AI-ism vocabulary: Discover, Explore, Embrace, Unlock, Master, Dive into, Learn the secrets of, Unleash
